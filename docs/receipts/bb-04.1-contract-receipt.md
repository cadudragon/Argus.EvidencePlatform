# Contract Receipt

- contract_id: `BB-04.1`
- title: `Encrypted FCM Command Envelope`
- stack_hint: `.NET`
- approval: `Aprovado`
- approved_at: `2026-04-10`

## Context

The backend must stop using plaintext FCM command payloads as the production contract and send Android-compatible encrypted command envelopes for on-demand screenshot commands.

## Approved Scope

- Extend `PUT /api/fcm-token` to accept and persist `fcmCommandKey`.
- Persist device command key metadata in `FcmTokenBinding`.
- Add EF Core migration for the new binding fields and the minimal per-case command policy table.
- Create ECDH P-256 + HKDF-SHA256 + AES-256-GCM envelope encryption.
- Preserve `POST /api/device-commands/screenshot` while changing dispatch to encrypted FCM data.
- Add minimal per-case command policy with `streamStartFps` defaulting to `2`.
- Support internal command shapes for `screenshot`, `stream_start`, `stream_stop`, and `ping` without adding public stream endpoints.
- Add tests proving decrypt compatibility, GCM authentication failure on tamper, no plaintext `cmd` in FCM payload, and `dkid` sourced from the persisted device key.
- Update canonical docs.

## Explicitly Excluded

- Live/video implementation.
- Public endpoints for `stream_start`, `stream_stop`, or `ping`.
- UI/backoffice/billing/premium account management.
- FCM tickle plus private command fetch.
- mTLS.
- Z85, fixed padding, random JSON keys, or Ed25519 wrapper.
- Backend key rotation.
- Changes to the `BB-08` evidence read path.

## Error Semantics

- `/api/fcm-token` returns `400 Bad Request` for invalid or missing `fcmCommandKey`.
- `/api/fcm-token` keeps `410 Gone` for unknown or inactive devices.
- `/api/device-commands/screenshot` keeps `404 Not Found` for missing device or missing FCM binding.
- `/api/device-commands/screenshot` returns `409 Conflict` for bindings that exist but do not contain a command key.
- `/api/device-commands/screenshot` keeps `410 Gone` for invalidated Firebase tokens.
- `/api/device-commands/screenshot` keeps `503 Service Unavailable` for Firebase dispatch or encryption configuration failure.

## Approved Data Contracts

```json
{
  "deviceId": "android-0123456789abcdef",
  "fcmToken": "fcm-token-value",
  "fcmCommandKey": {
    "alg": "ECDH-P256",
    "kid": "device-ecdh-0123456789abcdef",
    "publicKey": "base64url-spki-public-key"
  }
}
```

Final FCM data payload:

```json
{
  "enc": "1",
  "alg": "ECDH-P256-HKDF-SHA256+A256GCM",
  "kid": "<backend-key-id>",
  "dkid": "<device-command-key-id>",
  "iv": "<base64url-no-padding-12-byte-iv>",
  "ct": "<base64url-no-padding-ciphertext-plus-gcm-tag>"
}
```

Canonical AES-GCM AAD:

```json
{"enc":"1","alg":"ECDH-P256-HKDF-SHA256+A256GCM","kid":"<kid>","dkid":"<dkid>","iv":"<iv>"}
```

## Key Decisions

- Backend BB name: `BB-04.1`.
- Backend private key config format: `base64url-no-padding` PKCS#8 DER.
- Plaintext FCM fallback: explicit debug flag only, default `false`.
- Existing FCM binding without command key: `409 Conflict`.
- `stream_start.fps`: read from per-case command policy, default `2`; no public mutation endpoint in this BB.
